# Semantic Mapping Specification (UCA -> Predicate -> Rule)

## Mapping Contract
- Input: UCA entry (`domain`, `risk_type`, `mitre_tactic`, `trigger_event`, `hazard_ids`, `safety_constraint_ids`).
- Intermediate: predicate list (ordered conjunction).
- Output: AgentSpec rule text.

## Invariants
- Every UCA must map to exactly one trigger event.
- Every UCA must map to at least one predicate.
- `risk_type` must be compatible with `mitre_tactic`.
- `hazard_ids` must use `H{number}` format.
- `safety_constraint_ids` must use `SC-{2 digits}` format.
- Compiled rule must parse via `Rule.from_text`.

## Domain Mapping
- `code` domain:
  - Trigger usually `PythonREPL`.
  - Predicates from `rules/manual/pythonrepl.py`.
- `shell` domain:
  - Trigger usually `TerminalExecute`.
  - Predicates from `rules/manual/shell.py` plus OS checks.

## Enforcement Policy
- High-confidence hazards:
  - default `stop`.
- Ambiguous/high-business-risk hazards:
  - `user_inspection`.

## Traceability Contract
- Runtime audit keeps `rule_id` and best-effort `uca_id` reconstruction.
- Required lineage for each UCA:
  - `hazard_ids` -> why this UCA exists.
  - `safety_constraint_ids` -> which safety objective it enforces.
  - `risk_type` + `mitre_tactic` -> attack-model grounding.

## Determinism Rules
- Rule id generated by normalized `uca_id`.
- Predicate order preserved from `predicate_hints` or default mapping table.
- Templated output must be byte-stable for golden tests.
